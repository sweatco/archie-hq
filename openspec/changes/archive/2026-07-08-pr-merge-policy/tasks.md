## R2 — auto-merge arming

Revision R2 (design.md "## Revision R2") repositions the explicit `merge_pull_request` in a non-auto repo as *arming* auto-merge: approval means "merge this PR as soon as it is ready", the orchestrator merges an armed PR once GitHub reports it `clean`, and the explicit path no longer interprets mergeable state. All items below shipped on branch `forge/pr-merge-policy`.

- [x] R2.1 Add `merge_armed?: boolean` to `BranchState` (`src/types/task.ts`) — the persisted per-PR arm marker, alongside `merge_ready_notified`
- [x] R2.2 Rework `merge_pull_request` (`src/agents/tools.ts`, `createMergePRTool`): the non-auto branch prompts for **any open PR** (bailing only on a closed/merged PR), drops the `isMergeReadyPerGithub`/`approved` gate entirely, and frames the prompt as arming auto-merge ("merge automatically once all checks and required reviews pass") instead of merging now; the auto branch gates on `mergeableState === 'clean'` only (no `blocked` tolerance) and no longer calls `isMergeReadyPerGithub`; the tool description string rewritten to the arming contract. Suppression/supersede fork and slot persistence unchanged.
- [x] R2.3 Rework `Task.handleMergeApproval` (`src/tasks/task.ts`) to the merge-now-or-arm decision: after the atomic read-compare-clear, `open && clean` → merge now (completion finding); `open && not clean` → set `merge_armed` on every matching BranchState via the repositories → AttachedRepo[] → branch_states walk, append the decision finding "Auto-merge armed … will merge once checks pass", and persist **durably** with `save(true)`; closed/merged → arm nothing and record the real "nothing to merge" outcome. No `approved` check anywhere on the path (AC5).
- [x] R2.4 Add the armed bucket to the orchestrator (`src/connectors/github/merge.ts`, `runMergeCheck`): a PR whose BranchState carries `merge_armed` merges when `state === 'open' && mergeableState === 'clean'` — no `approved` floor, no `blocked` tolerance (`isMergeReadyPerGithub` deliberately not used here); `toMerge = autoMergeable ∪ armed` deduped by reference equality; armed PRs excluded from the held/`ready` bucket and its notification (AC1); `merge_armed` cleared when the PR is observed merged or closed
- [x] R2.5 Reset per-PR markers on branch reuse (`src/connectors/github/branch-state.ts`, `assignPrNumber`): clear both `merge_armed` and `merge_ready_notified` whenever a branch's `pr_number` changes, so a new PR opened on a reused branch never inherits a prior PR's arm — the authoritative reset (the close/merge webhook routes to `existing_task`, not a merge check, so its clear cannot be relied on)
